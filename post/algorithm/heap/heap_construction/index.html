Construct Binary Heap with Java


 

Construct Binary Heap with Java

二叉堆的Java实现
#Java     #algorithm     2017/Sep/25

前言 : 最近在看《算法导论》，从中感受到了算法世界的神奇，看完二叉堆后觉得实现起来应该不难，所以试着根据书中的伪代码用Java实现一遍。

0x00. 概念

1. 基本定义

二叉堆是一个数组，可近似看成完全二叉树，表示堆的数组A通常包含两个属性：A.length（数组长度）和A.heap-size（堆的有效元素）

private int[] heap;
private int heapSize;

public MyHeap(int[] array) {
    this.heap = array;
    this.heapSize = array.length;
}

2. 通过下标定位亲属节点

由于堆可看做完全二叉树，所以可以通过某节点的下标获取其左右子节点和父节点

private int left(int i) {
    return 2 * i + 1;
}

private int right(int i) {
    return 2 * i + 2;
}

private int parent(int i) {
    return (i - 1) / 2;
}

3. 堆的一些性质

  • 最大堆性质：A[PARENT[i] >= A[i]]
  • 最小堆性质：A[PARENT[i] <= A[i]]
  • 含有n个元素的堆的高为：小于等于lgn的最大整数
  • 当用数组表示存储了n个元素的堆时，叶节点下标分别为 (n/2) + 1, (n / 2) + 2, …, n（下标从１开始）

0x01. 维持堆的性质

使下标为i的节点所构成的堆成为最大堆

/**
 * 用于维持堆的性质
 * 在调用maxheapify时候可能出现left[i]和right[i]的二叉树嗾使最大堆
 * 但这时heap[i]可能小于其孩子
 * 这时通过递归调用自身实现“逐级下降”，从而使得当前子树代表的堆成为一个最大堆
 * 　
 * 时间复杂度：T(n)=O(lgn)
 * 对树高为h的节点，执行maxheapify的时间复杂度为O(h)
 * @param i
 */
private void maxheapify(int i) {
    int l = left(i);
    int r = right(i);
    int largest;

    if (l < heapSize && heap[l] > heap[i]) {
        largest = l;
    } else largest = i;

    if (r < heapSize && heap[r] > heap[largest])
        largest = r;

    if (largest != i) {
        // 若largest为i，说明以heap[i]为根节点的子树已是最大堆，方法结束
        int temp = heap[i];
        heap[i] = heap[largest];
        heap[largest] = temp;
        maxheapify(largest);
    }
}

同理可使下标为i的节点所构成的堆成为最小堆

注意到**当元素heap[i]比其孩子的值都大的时候，调用maxheapify该节点保持不变

0x02. 构建最大堆

每次调用 maxheapify 的时间复杂度为O(lgn)，buildMaxHeap 需要 O(n) 次这样的调用，所以估算该方法的总运行时间上界为: O(nlgn), 但更精确的计算得出的时间复杂度为 O(n)

private void buildMaxHeap() {
        // 构造一个最大堆
      for (int i = heapSize/2-1; i >= 0; i--) {
          maxheapify(i);
      }
    System.out.println("build Max-heap: " + Arrays.toString(heap));
}

public static void main(String[] args) {
    int[] data = new int[]{4, 1, 3, 2, 16, 9, 10, 14, 8, 7};
    MyHeap heap = new MyHeap(data);
    heap.buildMaxHeap();
}

console（同理亦可构建最小堆）

build Max-heap: [16, 14, 10, 8, 7, 9, 3, 2, 4, 1]

0x03. 堆排序

1. 思想

  • 将待排序的数组构造称最大堆
  • 将根节点和最后一个子节点的值互换
  • 通过”heapSize–“将数组的最后一个元素从堆中删除
  • 通过调用maxheapify(0)检查和确保剩下的堆是一个以heap[0]为根节点的最大堆，同时保证heap[0]是剩下堆中的最大元，为下一次交换做准备
  • 重复上述步骤直到i=1

2. 时间复杂度

每次调用buildMaxHeap()的时间复杂度为O(n)，而n-1次调用maxheapify(0)，每次的时间为O(lgn)，所以总时间复杂度:O(nlgn)

private void heapSort() {
    // 构建最大堆
    buildMaxHeap();
    // 将根节点和最后一个叶子节点进行交换
    for (int i = 1; i < heap.length; i++) {
        int temp = heap[heapSize - 1];
        heap[heapSize-1] = heap[0];
        heap[0] = temp;
        heapSize --;
        maxheapify(0);
    }
    System.out.println("sorted: " + Arrays.toString(heap));
}

public static void main(String[] args) {
    int[] data = new int[]{4, 1, 3, 2, 16, 9, 10, 14, 8, 7};
    MyHeap heap = new MyHeap(data);
    heap.heapSort();
}
