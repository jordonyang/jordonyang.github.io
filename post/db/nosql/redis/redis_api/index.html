Redis简介及安装

简介

Redis是一个开源的使用ANSI C语言编写、支持网络、可基于内存亦可持久化的日志型、Key-Value数据库，并提供多种语言的API。从2010年3月15日起，Redis的开发工作由VMware主持。从2013年5月开始，Redis的开发由Pivotal赞助。

特点：

  • 可以用来储存字符串，哈希结构，链表，集合，有序集合、
  • 具有“持久化”功能，用来存储

安装

官网：https://redis.io

cd /usr/local/src
$ wget http://download.redis.io/releases/redis-4.0.2.tar.gz
$ tar xzf redis-4.0.2.tar.gz
$ cd redis-4.0.2
$ make
make PREFIX=/usr/local/redis install

启动服务

cd /usr/local/redis
cp /usr/local/src/redis-4.0.2/redis.conf ./
cd ..
./bin/redis-server ./redis.conf

设置redis服务后台运行

vim redis.conf

将daemonize选项值改为yes

基本数据类型

通用键值操作

增加键

set keyname value

获取键的值

get keyname

查找键

keys pattern:

  • h?llo matches hello, hallo and hxllo
  • h*llo matches hllo and heeeello
  • h[ae]llo matches hello and hallo, but not hillo
  • h[^e]llo matches hallo, hbllo, … but not hello
  • h[a-b]llo matches hallo and hbllo

删除键

del keyname1 keyname2 keyname

返回的是删除的条数

清空数据库

flushdb

获取一个随机的键

randomkey

判断某个键是否存在

exists keyname

查询键的数据类型

type keyname

修改键名

rename oldkeyname newkeyname

注意：如果newkeyname已经存在，则原来newkey的值将会被覆盖

127.0.0.1:6379> set site "www.baidu.com"

2) "site"

OK
127.0.0.1:6379> get web
"www.baidu.com"
127.0.0.1:6379> keys *
1) "web"
  • nx: no exists
renamenx oldkeyname newkeyname

当newkeyname未存在时则可修改，修改成功返回1否则返回0

Redis的数据库

redis默认有16个数据库

# Set the number of databases. The default database is DB 0, you can select
# a different one on a per-connection basis using SELECT <dbid> where
# dbid is a number between 0 and 'databases'-1
databases 16

默认是在0号数据库上进行操作，可通过select [0-15]进行数据库的切换

select 1

将key在数据库间进行移动

move keyname dbno

如：

127.0.0.1:6379> keys *
1) "web"
2) "we"
127.0.0.1:6379> move we 1
(integer) 1
127.0.0.1:6379> keys *
1) "web"
127.0.0.1:6379> select 1
OK
127.0.0.1:6379[1]> keys *
1) "we"

key的有效期

查看key的有效期

127.0.0.1:6379> keys *
1) "web"
2) "we"
127.0.0.1:6379> ttl web
(integer) -1
127.0.0.1:6379> ttl wer
(integer) -2

过期的key返回-2，否则返回-1（秒数，-1表示永久有效）

设置key的有效期

如设置web的生命周期为10秒

127.0.0.1:6379> expire web 10
(integer) 1
127.0.0.1:6379> get web
(nil)

还可以毫秒为单位设置生命周期

127.0.0.1:6379> set web "www.baidu.com"
OK
127.0.0.1:6379> pexpire web 5000
(integer) 1
127.0.0.1:6379> ttl web
(integer) -2

将某个key设置为永久有效

127.0.0.1:6379> set web "www.baidu.com"
OK
127.0.0.1:6379> pexpire web 10000
(integer) 1
127.0.0.1:6379> persist web 
(integer) 1
127.0.0.1:6379> ttl web
(integer) -1

Redis字符串类型的操作

创建key同时指定生命周期，语法：

set key value [ex 秒数 | px 毫秒数]

linfeng:6379> set site www.baidu.com ex 10
OK
linfeng:6379> ttl site
(integer) 2
linfeng:6379> ttl site
(integer) -2

set key value [nx | xx]

linfeng:6379> keys *
(empty list or set)
linfeng:6379> set site "www.baidu.com" nx
OK
linfeng:6379> set web "www.google.com" xx
(nil)
linfeng:6379> keys *
1) "site"
linfeng:6379> get site
"www.baidu.com"
linfeng:6379> set site "www.google.com" xx
OK
linfeng:6379> keys *
1) "site"
linfeng:6379> get site
"www.google.com"
127.0.0.1:6379>
  • nx：当key不存在时执行操作
  • xx：当key存在时执行操作

多个key的设置和获取

  • mset key1 value1 key2 value2 … keyb valuen
  • mget key1 key2 … keyn
linfeng:6379> keys *
(empty list or set)
linfeng:6379> mset c controller d dao s service p po
OK
linfeng:6379> keys *
1) "s"
2) "d"
3) "p"
4) "c"
linfeng:6379> get s
"service"
linfeng:6379> get c
"controller"
linfeng:6379> mget s d p c
1) "service"
2) "dao"
3) "po"
4) "controller"

字符偏移替代

语法：setrange key offset value

linfeng:6379> keys *
(empty list or set)
linfeng:6379> set greet hello
OK
linfeng:6379> setrange greet 2 xx
(integer) 5
linfeng:6379> get greet
"hexxo"
linfeng:6379> setrange greet 6 !
(integer) 7
linfeng:6379> get greet
"hexxo\x00!"

注意：如果偏移量>字符串长度，则空缺处的字符由\x00补全

尾部追加值

语法：append key value

linfeng.1:6379> set hello "hello"
OK
linfeng:6379> append hello " world"
(integer) 11
linfeng:6379> get hello
"hello world"

获取子串

语法：getrange key start end，获取[start,end]范围的值

linfeng:6379> set word "fredistory"
OK
linfeng:6379> getrange word 1 5
"redis"

从右往左切掉某段连续的字符串

linfeng:6379> set status working
OK
linfeng:6379> getrange status 0 -4
"work"

注意：

  • 对于字符串的下标，从左数从０开始，从右数从－１开始
  • start>length，则返回空字符窜
  • end>length，则截取至字符串结束
  • 如果start所处的位置在stop右边，则返回空字符串

获取key的旧值同时设置新值

语法：getset key value

linfeng:6379> set status "coding"
OK
linfeng:6379> getset status "running"
"coding"
linfeng:6379> getset status "bathing"
"running"
linfeng:6379> getset status "sleeoing"
"bathing"

数值的增减

每次增减１

linfeng:6379> set age 20
OK
linfeng:6379> get age
"20"
linfeng:6379> incr age
(integer) 21
linfeng:6379> decr age
(integer) 20
linfeng:6379> type age
string

每次增减指定的数值

linfeng:6379> get age
"20"
linfeng:6379> incrby age 3
(integer) 23
linfeng:6379> decrby age 2
(integer) 21

进行浮点数值的增加

linfeng:6379> get age
"21"
linfeng:6379> incrbyfloat age 0.5
"21.5"

位运算

ASCII

A 65 0100 0001
a 97 0110 0001

将一个大写字母变成小写只需将第三位数改为１

linfeng:6379> set char A
OK
linfeng:6379> setbit char 2 1
(integer) 0
linfeng:6379> get char
"a"

注意

  • 如果offset过大，则会在中间填充０
  • 最大值字符串大小为512M

位操作

语法： bitop operation destkey key [key …]

linfeng:6379> bitop or char char lower

rpoplpush source destination

把source的尾部元素弹出放到destination的头部并返回该单元值

场景：task,back双链表完成安全队列

业务逻辑：

  • rpoplpush task back
  • 接收返回值，并作业务处理
  • 若成功，pop back删除任务；若失败，下次从back表中取任务
127.0.0.1:6379> rpush task a b c d
(integer) 4
127.0.0.1:6379> rpoplpush task back
"d"
127.0.0.1:6379> lrange task 0 -1
1) "a"
2) "b"
3) "c"
127.0.0.1:6379> lrange back 0 -1
1) "d"

brpop|blpop key [key …] timeout

等待弹出的key的尾/头元素，timeout为等待超时时间，若timeout为0则一直等待

场景：长轮询Ajax，在线聊天（读数据发数据）

message为空，设置等待３０秒，在这期间如果有数据读入，则将之弹出

zinterstore destination numkeys key [key 1,key2,…,keyn]|[WEIGHTS weight] |[AGGRIGATE SUM|MIN|MAX]

求key1,key2,…,keyn的交集，key1,key2,…的权重分别为weight1,weight2,…，numkeys表示作交集运算的key的个数聚合方法用sum,max.min,聚合结果保存在destination

中,返回值是destination中元素的个数

127.0.0.1:6379> zadd li 4 cat 3 dog 5 horse 7 duck
(integer) 4
127.0.0.1:6379> zadd liu 5 cat 2 dog 4 horse 4 donkey
(integer) 4
127.0.0.1:6379> zinterstore result 2 li liu 
(integer) 3
127.0.0.1:6379> zrange result 0 -1 withscores
1) "dog"
2) "5"
3) "cat"
4) "9"
5) "horse"
6) "9"

可见默认的聚合函数为SUM，当然还可指定其他函数

127.0.0.1:6379> zadd li 4 cat 3 dog 5 horse 7 duck
(integer) 4
127.0.0.1:6379> zadd liu 5 cat 2 dog 4 horse 4 donkey
(integer) 4
127.0.0.1:6379> zinterstore maxresult 2 li liu aggregate max
(integer) 3
127.0.0.1:6379> zrange maxresult 0 -1 withscores
1) "dog"
2) "3"
3) "cat"
4) "5"
5) "horse"
6) "5"
127.0.0.1:6379> zinterstore minresult 2 li liu aggregate min
(integer) 3
127.0.0.1:6379> zrange minresult 0 -1 withscores
1) "dog"
2) "2"
3) "cat"
4) "4"
5) "horse"
6) "4"

当然也可以指定key的权重

127.0.0.1:6379> zadd li 4 cat 3 dog 5 horse 7 duck
(integer) 4
127.0.0.1:6379> zadd liu 5 cat 2 dog 4 horse 4 donkey
(integer) 4
127.0.0.1:6379> zinterstore result 2 li liu weights 2 1 
(integer) 3
127.0.0.1:6379> zrange result 0 -1 withscores
1) "dog"
2) "8"
3) "cat"
4) "13"
5) "horse"
6) "14"

这里的score是一共卖出了多少个动物，weight可以是单个动物的价格，这样得出的sum为总收入

同理还有用于并集运算的zunionstore

zunionstore destination numkeys key [key …][WEIGHTS weight] [AGGREGATE SUM|MIN|MAX]

哈希结构

hset key field value

把key中的field域的值设为value，若没有field直接添加否则覆盖原field的值

127.0.0.1:6379> hset user name Mike
(integer) 1
127.0.0.1:6379> hset user age 20
(integer) 1
127.0.0.1:6379> hset user gender male
(integer) 1

hmset key field value [field value …]

创建key同时设置多个field的值

127.0.0.1:6379> hmset user1 name Daisy age 21 gender female
OK

hgetall key

获取key的所有field及其对应的值

127.0.0.1:6379> hgetall user
1) "name"
2) "Mike"
3) "age"
4) "20"
5) "gender"
6) "male"

hget key field

返回key中field对应的值

事务及锁应用

基本命令

  • multi：开启事务
  • discard：撤销
  • exec：执行
  • watch：监视
  • unwatch：取消对所有的key的监视

Ｄｅｍｏ１：模拟一个银行转账的过程

127.0.0.1:6379> set li 600
OK
127.0.0.1:6379> set wang 1000
OK
127.0.0.1:6379> multi
OK
127.0.0.1:6379> decrby wang 300
QUEUED
127.0.0.1:6379> incrby li 300
QUEUED
127.0.0.1:6379> exec
1) (integer) 700
2) (integer) 900
127.0.0.1:6379> mget wang li
1) "700"
2) "900"

可见在事务过程中，先将要执行的语句加入到队列中，若无错误则在确定exec后在执行，这样保证命令的原子性

注意事务出错主要有两种

  • 语法有问题，这时事务中所有语句都不执行
127.0.0.1:6379> mget wang li
1) "700"
2) "900"
127.0.0.1:6379> multi
OK
127.0.0.1:6379> decrby wang 200
QUEUED
127.0.0.1:6379> dsadasd
(error) ERR unknown command 'dsadasd'
127.0.0.1:6379> exec
(error) EXECABORT Transaction discarded because of previous errors.
127.0.0.1:6379> mget wang li
1) "700"
2) "900"
  • 语句本身没有问题，但适用对象有问题，这时只执行正确的命令
127.0.0.1:6379> mget wang li
1) "700"
2) "900"
127.0.0.1:6379> multi
OK
127.0.0.1:6379> decrby wang 200
QUEUED
127.0.0.1:6379> sadd li longlong
QUEUED
127.0.0.1:6379> incrby li 200
QUEUED
127.0.0.1:6379> exec
1) (integer) 500
2) (error) WRONGTYPE Operation against a key holding the wrong kind of value
3) (integer) 1100
127.0.0.1:6379> mget wang li
1) "500"
2) "1100"

discard，取消执行事务，严格意义上不算回滚，只是将任务队列清空,一旦队列中命令已经执行，则无法取消

127.0.0.1:6379> mget wang li
1) "500"
2) "1100"
127.0.0.1:6379> multi
OK
127.0.0.1:6379> incrby wang 100
QUEUED
127.0.0.1:6379> decrby li 100
QUEUED
127.0.0.1:6379> discard
OK
127.0.0.1:6379> mget wang li
1) "500"
2) "1100"

PUBSUB CHANNELS [pattern]

列出当前的活跃频道。

活跃频道指的是那些至少有一个订阅者的频道，订阅模式的客户端不计算在内。

pattern 参数是可选的：

  • 如果不给出 pattern 参数，那么列出订阅与发布系统中的所有活跃频道。
  • 如果给出 pattern 参数，那么只列出和给定模式 pattern 相匹配的那些活跃频道。

复杂度： O(N) ， N 为活跃频道的数量（对于长度较短的频道和模式来说，将进行模式匹配的复杂度视为常数）。

返回值： 一个由活跃频道组成的列表。

# client-1 订阅 news.it 和 news.sport 两个频道

client-1> SUBSCRIBE news.it news.sport
Reading messages... (press Ctrl-C to quit)
1) "subscribe"
2) "news.it"
3) (integer) 1
1) "subscribe"
2) "news.sport"
3) (integer) 2

# client-2 订阅 news.it 和 news.internet 两个频道

client-2> SUBSCRIBE news.it news.internet
Reading messages... (press Ctrl-C to quit)
1) "subscribe"
2) "news.it"
3) (integer) 1
1) "subscribe"
2) "news.internet"
3) (integer) 2

# 首先， client-3 打印所有活跃频道
# 注意，即使一个频道有多个订阅者，它也只输出一次，比如 news.it

client-3> PUBSUB CHANNELS
1) "news.sport"
2) "news.internet"
3) "news.it"

# 接下来， client-3 打印那些与模式 news.i* 相匹配的活跃频道
# 因为 news.sport 不匹配 news.i* ，所以它没有被打印

redis> PUBSUB CHANNELS news.i*
1) "news.internet"
2) "news.it"

PUBSUB NUMSUB [channel-1 … channel-N]

返回给定频道的订阅者数量，订阅模式的客户端不计算在内。

复杂度： O(N) ， N 为给定频道的数量。

返回值：一个多条批量回复（Multi-bulk reply），回复中包含给定的频道，以及频道的订阅者数量。格式为：频道 channel-1 ， channel-1 的订阅者数量，频道 channel-2 ， channel-2 的订阅者数量，诸如此类。回复中频道的排列顺序和执行命令时给定频道的排列顺序一致。不给定任何频道而直接调用这个命令也是可以的，在这种情况下，命令只返回一个空列表。

# client-1 订阅 news.it 和 news.sport 两个频道

client-1> SUBSCRIBE news.it news.sport
Reading messages... (press Ctrl-C to quit)
1) "subscribe"
2) "news.it"
3) (integer) 1
1) "subscribe"
2) "news.sport"
3) (integer) 2

# client-2 订阅 news.it 和 news.internet 两个频道

client-2> SUBSCRIBE news.it news.internet
Reading messages... (press Ctrl-C to quit)
1) "subscribe"
2) "news.it"
3) (integer) 1
1) "subscribe"
2) "news.internet"
3) (integer) 2

# client-3 打印各个频道的订阅者数量

client-3> PUBSUB NUMSUB news.it news.internet news.sport news.music
1) "news.it"    # 频道
2) "2"          # 订阅该频道的客户端数量
3) "news.internet"
4) "1"
5) "news.sport"
6) "1"
7) "news.music" # 没有任何订阅者
8) "0"

PUBSUB NUMPAT

返回订阅模式的数量。

注意，这个命令返回的不是订阅模式的客户端的数量，而是客户端订阅的所有模式的数量总和。

复杂度： O(1) 。

返回值： 一个整数回复（Integer reply）。

# client-1 订阅 news.* 和 discount.* 两个模式

client-1> PSUBSCRIBE news.* discount.*
Reading messages... (press Ctrl-C to quit)
1) "psubscribe"
2) "news.*"
3) (integer) 1
1) "psubscribe"
2) "discount.*"
3) (integer) 2

# client-2 订阅 tweet.* 一个模式

client-2> PSUBSCRIBE tweet.*
Reading messages... (press Ctrl-C to quit)
1) "psubscribe"
2) "tweet.*"
3) (integer) 1

# client-3 返回当前订阅模式的数量为 3

client-3> PUBSUB NUMPAT
(integer) 3

# 注意，当有多个客户端订阅相同的模式时，相同的订阅也被计算在 PUBSUB NUMPAT 之内
# 比如说，再新建一个客户端 client-4 ，让它也订阅 news.* 频道

client-4> PSUBSCRIBE news.*
Reading messages... (press Ctrl-C to quit)
1) "psubscribe"
2) "news.*"
3) (integer) 1

# 这时再计算被订阅模式的数量，就会得到数量为 4

client-3> PUBSUB NUMPAT
(integer) 4

UNSUBSCRIBE [channel [channel …]]

指示客户端退订给定的频道。

如果没有频道被指定，也即是，一个无参数的 UNSUBSCRIBE 调用被执行，那么客户端使用SUBSCRIBE命令订阅的所有频道都会被退订。在这种情况下，命令会返回一个信息，告知客户端所有被退订的频道。

  • 可用版本：

= 2.0.0

  • 时间复杂度：

O(N) ， N 是客户端已订阅的频道的数量。

  • 返回值：

这个命令在不同的客户端中有不同的表现。
